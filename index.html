Inleiding In dit verslag ga ik mij verdiepen in software die van belang is voor mijn toekomstige beroep als Cloud Engineer. Ik ga mijzelf orienteren op de arbeidsmarkt en bepaal welke software het meest relevant is om mee te werken als Cloud Engineer. Opdracht 1 Wat doet een Cloud Engineer Een Cloud engineer is een specialist binnen de IT die verantwoordelijk is voor het ontwerpen, bouwen, beheren en optimaliseren van cloudomgevingen. De werkzaamheden van een cloud engineer bestaan onder andere uit: • Opzetten en beheren van cloudinfrastructuur (servers, databases, netwerken, storage) • Automatiseren van processen via scripts en infrastructure as code • Monitoren en optimaliseren van prestaties en kosten van cloudomgevingen • Implementeren van beveiligingsmaatregelen en zorgen voor compliance • Ondersteunen van ontwikkelteams bij het deployen van applicaties • Troubleshooten van problemen in cloudomgevingen. Criteria voor het kiezen van relevante software Bij het kiezen van software waarin ik mij wil verdiepen hanteer ik de volgende criteria: • Relevantie op de arbeidsmarkt • Breed inzetbaar • Toekomstbestendigheid • Integratie met andere tools • Beschikbaarheid van leermaterialen Overzicht van relevante softwarepakketten Op basis van vacatures en forums ben ik erachter dat cloud engineers vaak in aanraking komen met: • Cloudplatforms als AWS, Azure, Google Cloud • Infrasructure as code (IaC) op Terraforms, AWS, Cloudformation, Pulumi • Contairesitatie als Docker, Kubernetes • CI/CD als Jenkins, GitLab CI/CD, GitHub, Actions, Azure DevOps • Monitoring & Logging als Prometheus, Grafana, Cloudwatch, Splunk • Scripting & Automatisering met Python, Bash Powershell Selectie van software Na het onderzoeken van de vacatures en forums blijkt dat vooral de volgende softwarepakketten vaak genoemd worden en essentieel zijn voor cloud engineers: • Azure – Het meest gebruikte cloudplatform • Terraforms – De populairste IaC-tool voor cloudomgevingen • Docker en Kubernetes – een standaard voor containerisatie • Jenkins en GitLab CI/CD – veelgebruikte CI/CD oplossing Gemaakte keuze en motivatie Ik kies ervoor om mij te verdiepen in Microsoft Azure, omdat je met een brede Azure kennis jezelf een sterke positie geeft op de arbeidsmarkt. Hierbij leer ik niet alleen de basics (Fundamentals), maar ook specifiek het beheren van virtuele omgevingen en het gebruiken van cloud resorces. Argumentatie Mijn keuze voor Miscrosoft Azure is gebasseerd op drie sterke argumenten: • Marktaandeel en vraag: In Nederland zitten veel bedrijven gevestigd die gebruik maken van de Microsoft-Stack (Office 365, Windows server). De overstap naar Azure is voor hen een logische stap, om onder andere op kosten te kunnen besparen. Hierdoor is de vraag naar ICT’ers met Microsoft Azure kennis aan het groeien. • Integratie mogelijkheden: Azure biedt een naadloze integratie met tools die ik in de toekomst ook zal moeten beheersen, zoals Terraform (voor Infrastructure as Code) en GitHub actions (voor CI/CD) • Toekomstbestendigheid: Microsoft investeert in cloud-innovatie en AI (zoals Azure openAI). Door mijzelf nu in Azure te verdiepen, bouw ik kennis op die de komende tien jaar relevant blijft. Proces van keuze Het proces om tot deze keuze te komen verliep in drie stappen: • Marktonderzoek: Ik heb verschillende vacatures bekeken, hieruit bleek dat de regio waarin ik wil werken, de vraag naar Azure-specialisten groter is dan die naar AWS of Google Cloud. • Vergelijking: Ik heb gekeken naar de aangeboden leerstof van verschillende platforms. Hoewel AWS ook interessant is, vind ik de documentatie en structuur van het Microsoft Learn platform beter bij mij passen. • Toekomstvisie: Ik heb mijn persoonlijke interesses naast de functies van de pakketten gelegd. De focus van Azure op hybride cloud-oplossingen spreekt mij erg aan. Overtuiging van opdrachtgever/leidinggevende Door mij te specialiseren in Microsoft Azure, word ik tijden mijn stage direct inzetbaar bij een breed scala aan bedrijven. In plaats van alleen algemene IT-kennis bied ik mijn stage bedrijf straks een specifieke expertise is een schalen en beveiligen van cloud-omgevingen. Opdracht 3 Eerdere kennis en vaardigheden Tijdens mijn opleiding aan het ROCvA heb ik al gewerkt met basis IT-concepten en on-premise servers. • Overeenkomsten: De basisprincipes zoals computer resources, opslag en netwerken blijven gelijk. Een server in Azure heeft nog steeds een IP-adrese en een besturingsysteem nodig, net als een fysieke server. • Verschillen: Het grootste verschil is het Capital Expenditure (CapEX) vs Operational Expenditure (OpEx) model. Bij on-premise koop je hardware (CapEx), bij Azure betaal je voor wat je verbruikt (CopEx). Ook het concept van het ‘shared responsibility’ is nieuw. Microsoft beheert de fysieke beveiliging, ik de data.
